The original circuit for PICPROG was quickly thrown together on a piece of "strip board" or "vero-board". This is the circuit which is used in the ugly looking prototype programmer shown on the main page. Follow this link to view the GIF image of the original circuit.

The current version of the circuit has had a number of improvements. Some of these, including the new layout, were provided by Valter Gruntar. I have made a few more improvements of my own and present the current version here. I have tried to keep the image resolution high enough so that no detail is lost.

The IVEX Windraft file for the current circuit is available. download

There are 200 and 350 pin versions of Windraft / Winboard at hobbyist type prices at NTE Electronics, Inc.

Parts List
Resistors
R1, R7 1K
R2, R5, R6, R9, R10 10K
R3 56K
R4 3K3 (3.3K)
R8 2K7
R11 4K7
Capacitors
C1 220uF / 35v Electrolytic
C2, C3, C4, C7, C8 100n (0.1uF)
C5, C6 1uF / 16v Electrolytic
Transistors/Diodes
Q1, Q2 BC557 or DS557 or any other general purpose PNP silicone transistor.
D1 IN4007 or IN4001 (rev. polarity protection)
D2 IN4148, IN914 or any general purpose signal diode.
Integrated Circuits
U1 UA78L08 - 3 pin 8 volt positive voltage regulator (TO-92 (small) package)
U2 UA78L05 - 3 pin 5 volt positive voltage regulator (TO-92 (small) package)
U3(A,B,C,D,E,F) DM7407N (7407 of any brand) HEX Buffer with open collector outputs. Note: could be replaced by diodes with cathodes toward LPT interface BUT damage to LPT port would be more likely.
L.E.D.s
LED1, LED2 5mm Red light emitting diode. Replace with whatever you like.
Connectors
J1, J3 Use whatever type of connector makes sense to you and is readily available.
J2 As above but ideally a Z.I.F. socket (levered Zero Insertion Force) to accept the type of PIC you are programming (18 pin in the case of a PIC16C84 DIL package)
NOTES:
Component Values
Resistances are in Ohms. K = Kilo (x1000). nKd = n point d Kilo Ohms,
Resistor are 1/8 or 1/4 watt minimum 10% tolerance.

Capacitance is as indicated. 1n = 1 nano-Farad, 1u = 1 micro Farad,
electrolytic capacitor voltage is not critical but must be rated at least the supply voltage where connected.

Power Supply
Power for this programmer can be provided in any fashion available. The circuit uses a maximum 60mA accounting for both LED's running and absolute maximum power consumption by PIC being programmed. Obviously if you use the programmer's power supply to power a host circuit while performing in-circuit programming you willneed to take the host circuit consumption into account.

For stand-alone programming two 9-Volt "transistor radio" batteries connected in series should last for months.

Following is a summary of the changes made from the original circuit (from memory) and why:

  • The pull-up resistors for the PIC programming pins were originally connected to the permanently 'on' 5 volt supply rail. This meant these pins were effectively 'live' during chip insertion and removal. Valter was the first to recognize the design error and corrected it in his version of the circuit.
     
  • The 78L12 voltage regulator from the original circuit has been replaced by a 78L08 connected 'on top' of the 5 volt regulator's output. Apart from being more efficient this also gives one more volt to make up for the voltage drop produced by the Vpp supply switching transistor. Microchip state 12v-14vdc for the programming voltage. My original circuit only produce 11.4 volts (approx.) It never failed to program successfully but was out of spec. This is another 'Valter' enhancement.
     
  • Extra supply decoupling capacitors have been added at the output of each switch transistor to filter noise produce in the silicone junction. This one is more a case of 'proper' design than a necessity. No one was ever harmed by doing something right! :)
     
  • An extra decoupling capacitor was added to the PIC chip power supply right up close to it's socket. Again - proper design.
