Phone cards in CIS

The majority of payphones in CIS use smartcars based on SLE4406-compliant chip. The chip has 128 bit of internal memory. 64 bits used as card identifier, 40 bits organize credits (card units) counter, 24 bits are unused. The counter range is 0..32767 units. The chip only allows do decrease counter value. Card memory can be easily read using LPT printer port. The card reader program is here:

Card RESET pin must be connected to LPT D0 pin
Card CLK pin must be connected to LPT D1 pin
Card DATA pin must be connected to LPT BUSY pin
Card GND must be connected to one of LPT GND pins.
Card Vcc pin must be connected to LPT pin D7 or to external 5V source
All ISO 7816 parameters (pinout, protocol, time diagrams) are described here. Similar description is also available in Phrack magazine issue 48 vol.10, but that information is a bit inaccurate.

How do wire detectors work

I know wire detector devices of 2 types. First type is simple metal detector which detects metal presence on card surface. This type of wire detectors is not widely used because it is rather simply for phreakers to pass it. Second type of wire detectors uses high frequency to detect wires presence. HF signal is mixed to one of cards pins (usually to power supply pin). Sensor detects presence of that frequency on the card's surface. This type of wire detectors is able to detect very thin wires and could hardly be passed through.
If you know any other type of wire detectors, please inform me.
